# cpp-parallel

Репозиторий задания "Синхронизация нескольких потоков исполнения через очередь сообщений на C++" (LEARNING_CENTER-106)

Выполнила: Мухачева Полина

## Запуск GTests
### Из bash-скрипта
В корне проекта выполнить команду
```
bash GTestsStart.sh
```
### Вручную
1. Удалить директорию build/, если она уже существует
```
rm -rf build
```
2. Создать build/ и запустить из нее сборку проекта
```
mkdir build
cd build
cmake ..
make
```

3. Запустить проект
```
./test/test_device
```

## Запуск CLI
### Из bash-скрипта
В корне проекта выполнить команду
```
bash CLIStart.sh
```
### Вручную
1. В корне выполнить
```
make
```
2. В корне выполнить
```
./run
```

## Работа приложения

При запуске на экран выводятся инструкции по запуску нескольких сценариев работы. Необходимо выбрать нужный сценарий, введя значение, требуемое инструкцией. При некоторых из сценариев работы приложение требует ввести дополнительные параметры -- они вводятся в виде натурального числа.

В режиме корректной работе устройства его остановка из консольного приложения не предусмотрена. Устройства работают на протяжении 100 итераций и затем останавливаются.
Если необходимо выполнить выход из программы раньше, это можно сделать через прерывание Ctrl+C.
